## Design Context

### Users

Maintainers sign in through Cloudflare Access to see the services they can use
and test them in the playground. Administrators manage a much larger service
catalog, connect services to access grants, inspect effective user access, and
monitor grant health.

### Brand Personality

Calm, sharp, sophisticated, and operational. ClawRouter should feel like a
well-run control room, not a developer toy or a generic SaaS dashboard.

### Aesthetic Direction

Use a desktop-first light workbench with ink-like text, paper-white working
surfaces, compact controls, sharp one-pixel boundaries, and clear green, red,
blue, and yellow state signals. Use a humanist system UI font for interface copy
and the system monospace stack sparingly for metadata. Avoid purple gradients, floating card grids,
oversized titles, excessive padding, and decorative technical imagery.

### Design Principles

1. Treat a service as the access-control unit; models and routes are supporting
   details.
2. Keep scanning and repeated admin actions dense, predictable, and fast.
3. Make prompt composition and response review the primary playground flow.
4. Keep request JSON, curl, and transport details hidden until explicitly
   inspected.
5. Show effective access and readiness outcomes, not just configuration inputs.
6. Use color to communicate state, never as decoration alone.
